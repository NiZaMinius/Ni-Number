# ni-number &nbsp;[![Crates.io](https://img.shields.io/crates/v/ni-number)](https://crates.io/crates/ni-number) [![Docs.rs](https://docs.rs/ni-number/badge.svg)](https://docs.rs/ni-number) [![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

Вычисление константы Ни (η_ν) — квантовой константы рассеяния энергии — на Rust с произвольной точностью и несколькими сменными бэкендами.

```
η_ν = 1.88937666040491913115597775087642096081019761538215...
```

## Концепция квантовой константы рассеяния энергии (число Ни)

**Автор гипотезы:** NiZaMinius

### 1. Определение и фундаментальный смысл

Квантовая константа рассеяния энергии $\eta_{\nu}$ **(число Ни)** — фундаментальная математическая величина, определяющая полную амплитуду затухания калибровочного поля на суб-планковских расстояниях.

Значение числа Ни строго фиксировано математически и приблизительно равно **1.8893766...**

Современная физика постулирует существование скрытых, компактифицированных пространственных измерений (многообразия Калаби–Яу). Согласно этой концепции, при сближении частиц на расстояния меньше размера атомного ядра часть виртуальных бозонов (переносчиков взаимодействия) рассеивается в этот скрытый объём. Число Ни — точная мера доли энергии, покидающей трёхмерную брану.

---

### 2. Математическая структура константы

Константа определяется рядом, нормированным по всем модам спектра Калуцы–Клейна:

$$\eta_{\nu} = \sum_{n=1}^{\infty} \frac{\pi^n}{n! \cdot 2^{n^2}}$$

Индекс $n$ (1, 2, 3, ...) обозначает не порядковый номер скрытого измерения (все измерения существуют одновременно), а **номер квантовой моды** (уровень энергии). Подобно тому как гитарная струна порождает набор гармоник, кванты поля резонируют внутри единого многомерного многообразия на дискретных частотах. Бесконечный ряд суммирует все возможные квантовые состояния утечки энергии.

Каждый элемент формулы имеет строгое обоснование в дифференциальной геометрии и теории струн:

#### 2.1. Геометрия объёмных резонансов (числитель $\pi^n$)

Степени $\pi$ определяют геометрический фактор многомерности. Растущий показатель отражает расширение площади волнового фронта по мере проникновения энергии в новые координаты на каждой последующей гармонике.

#### 2.2. Топологическая симметрия (факториал $n!$)

Появление факториала — прямое следствие дифференциальной геометрии комплексных кэлеровых многообразий (пространств Калаби–Яу).

Для вычисления физического объёма такого многообразия необходимо проинтегрировать его фундаментальную форму объёма, построенную из кэлеровой 2-формы ($\omega$). Точная формула объёма:

$$V = \int_{M} \frac{\omega^n}{n!}$$

Из-за антикоммутативности внешнего произведения (учитывающей ориентацию осей) умножение порождает огромное число перекрывающихся перестановок координат. Математика требует делить интеграл на число этих перестановок ($n!$). Таким образом, факториал в числе Ни естественным образом **сворачивает избыточные симметрии** компактифицированного пространства, не давая объёму и утекающей энергии расти бесконечно.

#### 2.3. Квантовое блокирование (основание $2$ и показатель $n^2$)

В физике непрерывных процессов затухание обычно описывается числом Эйлера $e$. Однако на суб-планковских масштабах пространство дискретно.

В матричных моделях M-теории (например, BFSS) взаимодействия между бранами описываются матрицами $n \times n$, где $n$ — квантовое число моды. Размер матрицы $n^2$ определяет число степеней свободы квантового канала. Поскольку на фундаментальном уровне квантовая информация бинарна (кубит: связь есть или нет, спин вверх или вниз), **два** является основанием состояния.

Полное число микросостояний равно $2^{n^2}$. Использование $e$ означало бы применение классической статистики к квантовой гравитации. Основание 2 — сигнатура фундаментальной информационной природы пространства. Квадратичный рост числа связей вызывает лавинообразное сопротивление на высоких частотах, обеспечивая **сверхбыструю сходимость** ряда.

### 3. Модификация законов взаимодействия на ультрамалых расстояниях

Согласно данной гипотезе, классические законы обратного квадрата (Ньютон и Кулон) нарушаются на суб-планковских расстояниях, что предсказывается моделями с дополнительными измерениями (например, теорией Рэндалл–Сандрум).

Число Ни ($\eta_{\nu}$) — не просто коэффициент, вдвое ослабляющий силу в макромире. Оно выступает **динамическим интегральным коэффициентом** в самом уравнении поля, служа амплитудой перехода между браной (трёхмерным миром) и балком (пространством высших измерений).

Модифицированный закон силы:

$$F(r) = \frac{G \cdot m_1 m_2}{r^2} \cdot \frac{1}{1 + \eta_{\nu} \cdot \left(\frac{R_{\mathrm{Planck}}}{r}\right)^d}$$

где $R_{\mathrm{Planck}}$ — планковская длина, $r$ — расстояние, $d$ — эффективное число скрытых измерений.

**Физический смысл:**

| Режим | Физика |
|---|---|
| $r \gg R_{\mathrm{Planck}}$ (макромир) | Планковская доля → 0, знаменатель → 1, классический закон восстанавливается |
| $r \to R_{\mathrm{Planck}}$ (микромир) | Активируется многомерная физика; η_ν задаёт «пропускную способность» квантового портала; переносчики взаимодействия утекают в геометрические поры многообразия, вызывая резкое падение регистрируемой силы |

## Установка

Добавьте в `Cargo.toml`:

```toml
# По умолчанию — чистый Rust, работает везде, без системных зависимостей
[dependencies]
ni-number = "0.2"
```

### Бэкенды

`ni-number` поставляется с тремя сменными бэкендами:

| Feature | Бэкенд | Системные зависимости | Точность |
|---|---|---|---|
| `backend-dashu` | чистый Rust **(по умолчанию)** | нет | произвольная |
| `backend-rug` | GNU MPFR | GMP + MPFR | произвольная |
| `backend-f64` | нативный f64 | нет | ~15 цифр |

**По умолчанию (чистый Rust, рекомендуется):**
```toml
[dependencies]
ni-number = "0.2"
```

**Максимальная производительность через GNU MPFR:**
```toml
[dependencies]
ni-number = { version = "0.2", default-features = false, features = ["backend-rug"] }
```

**Минимальный размер (embedded / WASM):**
```toml
[dependencies]
ni-number = { version = "0.2", default-features = false, features = ["backend-f64"] }
```

### Системные зависимости для `backend-rug`

Нужны только при явном включении `features = ["backend-rug"]`.

**Ubuntu / Debian**
```bash
sudo apt-get install libgmp-dev libmpfr-dev libmpc-dev
```

**macOS (Homebrew)**
```bash
brew install gmp mpfr libmpc
```

**Windows (MSYS2)**

1. Установите [MSYS2](https://www.msys2.org)
2. В терминале MSYS2 MinGW64:
```bash
pacman -S --needed base-devel mingw-w64-x86_64-toolchain m4 make mingw-w64-x86_64-gmp mingw-w64-x86_64-mpfr
```
3. Добавьте в PATH в Git Bash (`~/.bashrc`):
```bash
export PATH="/c/msys64/mingw64/bin:/c/msys64/usr/bin:$PATH"
```
4. Переключите тулчейн Rust:
```bash
rustup toolchain install stable-x86_64-pc-windows-gnu
rustup default stable-x86_64-pc-windows-gnu
```

## Использование

```rust
use ni_number::{NI_F64, ni_number_digits, ni_number, bits_for_digits};
use ni_number::backend::NiFloat;

fn main() {
    // Быстрая f64 константа — без вычислений, предвычисленная (~15 цифр)
    println!("η_ν ≈ {:.15}", NI_F64);

    // 100 десятичных цифр как String — результат кэшируется после первого вызова
    let s = ni_number_digits(100);
    println!("η_ν = {}", s);

    // Значение произвольной точности для дальнейших вычислений
    let eta = ni_number(bits_for_digits(500));
    println!("{}", eta.to_decimal_string(500));
}
```

```bash
# Бенчмарк высокой точности
cargo run --example high_precision --release
```

## API

| Элемент | Возвращает | Описание |
|---|---|---|
| `NI_F64` | `f64` | Предвычисленная константа двойной точности |
| `NI_F32` | `f32` | Предвычисленная константа одинарной точности |
| `NI_50_DIGITS` | `&str` | Первые 50 десятичных цифр, статическая строка |
| `ni_number(bits)` | float бэкенда | Полное значение произвольной точности, кэшируется |
| `ni_number_digits(n)` | `String` | Десятичная строка с `n` цифрами после точки, кэшируется |
| `ni_series(bits)` | `NiSeries` | Ленивый итератор по членам ряда |
| `bits_for_digits(n)` | `u32` | Битовая точность для `n` десятичных цифр |
| `clear_cache()` | `()` | Освободить кэшированные значения из памяти |

## Производительность

Ряд сходится менее чем за 10 итераций при любом практическом уровне точности — работает даже на слабом железе. Результаты кэшируются после первого вызова — повторные вызовы с той же точностью мгновенны.

| Время (release, dashu) |
| Цифры  | dashu(series) | dashu (total) | rug (total) |
|--------|---------------|---------------|-------------|
| 100    | ~0.14 ms      | ~0.14 ms      | ~0.1 ms     |
| 1 000  | ~0.9 ms       | ~0.9 ms       | ~0.6 ms     |
| 5 000  | ~13.5 ms      | ~13.5 ms      | ~10.9 ms    |
| 10 000 | ~146 ms       | ~449 s        | ~52 ms      |

*dashu: только для серий. Для форматирования более 10 000 цифр на практике требуется. `backend-rug`.
